# 業務/送貨路線排程系統 — 規格書 v2

> 本版異動重點：系統拆分為「**業務模式**」與「**物流模式**」兩大流程，並新增物流的派遣單（訂單）作業。業務模式維持原設計，不做大幅更動。

## 1. 專案目的

提供公司業務與送貨人員使用的路線排程系統。App 開啟後可選擇以「業務」或「物流」身份操作：

- **業務模式**：業務人員自行勾選當日要拜訪的客戶，系統自動排出最佳化路線（沿用原設計，未更動）。
- **物流模式**：內勤人員先在網頁後台匯入派遣單（客戶＋送貨產品項目與數量），物流主管於 App 勾選當日實際配送客戶與優先客戶後，交由送貨人員產生/調整路線並配送。

## 2. 系統範圍

| 項目 | 說明 |
|---|---|
| 使用裝置 | 手機 App（業務人員／物流主管／送貨人員操作）＋ 網頁後台（資料維護、派遣單匯入） |
| 核心功能 | 拜訪/送貨路線最佳化排序；物流派遣單管理 |
| 主要使用者 | 業務人員、物流主管、送貨人員 |
| 管理者 | 內勤人員（透過網頁後台維護客戶、人員資料與派遣單） |

### 2.1 App 入口與角色

- App 啟動首頁提供兩個入口：**業務** / **物流**
- 人員資料新增「角色」欄位（業務人員／物流主管／送貨人員）。物流主管與送貨人員為互斥角色，實務上不會同一人擔任
- 選擇「物流」後，系統依該帳號的角色直接顯示對應畫面：
  - 「物流主管」角色 → 看到「派遣單勾選」功能
  - 「送貨人員」角色 → 看到「今日配送名單與路線」功能

## 3. 資料結構

### 3.1 客戶資料（未更動）

| 欄位 | 說明 |
|---|---|
| 優先客戶 | 是／否，預設「否」。業務模式沿用原邏輯；物流模式的優先客戶由物流主管於勾選當下另外標記（見 3.4） |
| 客戶編號 | 公司內部客戶代碼（例如 C001），需唯一不可重複 |
| 客戶名稱 | 客戶名稱 |
| 住址 | 完整地址，系統用此計算地圖座標與路線距離，並自動解析「縣市」 |
| 電話 | 純文字儲存，不用於路線計算 |

匯入方式：手動輸入 或 匯入 Excel（.xlsx）／CSV 檔案（需 UTF-8 編碼）。

### 3.2 人員資料（新增角色欄位）

| 欄位 | 說明 |
|---|---|
| 姓名 | 人員姓名 |
| 角色 | 業務人員／物流主管／送貨人員。「物流主管」與「送貨人員」為互斥角色（實務上不會同一人擔任），「業務人員」則可與其他角色並存 |
| 住家地址 | 作為預設出發地/目的地選項之一 |
| LINE 群組 | 單一值，固定對應該人員要推播路線通知的 LINE 群組（Token/Webhook） |

### 3.3 系統設定（未更動）

| 項目 | 說明 |
|---|---|
| 公司地址 | 系統層級固定值，全公司共用，作為出發地/目的地選項之一 |

### 3.4 派遣單／訂單資料（新增，僅物流模式使用）

派遣單欄位皆為**純文字欄位**，透過 CSV 匯入，彼此獨立不強制對應客戶主檔（若客戶編號恰好存在客戶主檔可供比對參考，但不影響匯入與路線計算）：

| 欄位 | 說明 |
|---|---|
| 派遣單編號 | 系統自動產生 |
| 送貨日期 | 該筆訂單預計配送日期 |
| 客戶編號 | 純文字欄位 |
| 客戶名稱 | 純文字欄位 |
| 住址 | 純文字欄位，系統用此計算地圖座標與路線距離 |
| 電話 | 純文字欄位，不用於路線計算 |
| 產品項目 | 純文字欄位，可為單一或多筆產品名稱（同一客戶多產品可用多列表示，不建立獨立產品主檔） |
| 數量 | 對應產品項目的數量 |
| 狀態 | 待處理／已勾選配送／已派送給送貨人員／已完成（系統依流程自動更新，見第 5 章） |
| 優先客戶標記 | 是／否，預設「否」。由物流主管在勾選當日配送清單時手動標記，僅對當筆派遣單生效，與客戶主檔無關 |

匯入方式：內勤人員於網頁後台以 CSV 批次匯入（需 UTF-8 編碼）。

**訂單異動（出發前可隨時新增／修改）**：送貨人員實際出發前，內勤人員可隨時於後台新增或修改派遣單內容（客戶、產品項目、數量、住址等），詳細通知機制見 5.4。

## 4. 業務模式（App 操作流程，未更動）

1. **選擇出發地**：公司 或 住家（住家自動帶入該人員的預設地址）
2. **選擇目的地**：公司 或 住家
3. **勾選今日客戶**：
   - 客戶清單依「縣市」分類，呈現可展開/收合的手風琴介面
   - 點擊縣市名稱展開，列出該縣市所有客戶（依客戶編號排序）
   - 支援依客戶編號或名稱搜尋
   - 優先客戶（客戶主檔標記）會有標籤提示
4. **產生路線**：呼叫地圖 API 計算距離，優先客戶固定排在路線最前段，其餘依最短路徑排序
5. **路線結果畫面**：時間軸方式呈現，排序完成後可離線查看
6. **調整路線**：有網路時可重新勾選客戶或調整順序，即時重新排序
7. **開始導航**：串接地圖 App 導航至下一站
8. **分享到 LINE 群組**：推送完整路線內容至該人員固定綁定的 LINE 群組

## 5. 物流模式（新增）

### 5.1 派遣單建立（網頁後台，內勤人員操作）

1. 內勤人員於網頁後台批次匯入 CSV 派遣單，內容包含：送貨日期、客戶編號、客戶名稱、住址、電話、產品項目、數量（皆為純文字欄位，見 3.4）
2. 匯入後的派遣單狀態為「待處理」，依送貨日期歸類，供物流主管於 App 端查看
3. 可針對「今日」或「近期（未來數日）」的派遣單分別匯入與查詢

### 5.2 物流主管操作流程（App）

1. 物流主管登入 App，選擇「物流」→ 進入派遣單勾選畫面
2. 畫面列出「待處理」狀態的派遣單，依送貨日期分類，同一客戶多筆產品項目合併顯示
3. 物流主管勾選**今日實際要配送**的客戶（可能小於等於匯入清單中的全部客戶）
4. 針對勾選的客戶，物流主管可另外標記「優先客戶」（僅對本次配送生效，見 3.4）
5. 送出後，系統：
   - 將勾選客戶的派遣單狀態更新為「已勾選配送」
   - 呼叫地圖 API 依「優先客戶固定最前＋其餘最短路徑」規則，排出**預設路線順序**
   - 將今日配送名單（含產品項目、數量、優先標記、預設路線順序）指派給對應的送貨人員

### 5.3 送貨人員操作流程（App）

1. 送貨人員登入 App，選擇「物流」→ 看到物流主管派送的今日配送名單（含系統預排的路線順序）
2. **可像業務人員一樣自行調整重排**：
   - 可重新選擇出發地／目的地（公司或住家）
   - 可調整勾選的客戶或路線順序，系統即時重新呼叫最佳化引擎排序
3. 路線結果畫面：時間軸呈現（出發地 → 各配送點含產品項目、數量、優先標籤、與上一站距離 → 目的地），排序完成後可離線查看
4. 開始導航：串接地圖 App 導航至下一站
5. 分享到 LINE 群組：推送完整配送路線（含產品項目、數量）至該人員固定綁定的 LINE 群組
6. 配送完成後，派遣單狀態更新為「已完成」（不含簽收/打卡功能，僅作為狀態紀錄）

### 5.4 派遣單異動通知機制（新增）

送貨人員出發前，訂單內容仍可能隨時新增或修改（客戶、產品項目、數量、住址等），因此後台一有異動，系統須主動通知相關人員：

1. **「出發前」的定義**：送貨人員尚未於 App 按下「開始配送／導航」動作，視為出發前；一旦出發，當日路線鎖定，異動不再即時通知，改列入下一趟或次日派遣單處理
2. **通知對象與時機**：
   - 派遣單狀態為「待處理」（尚未經物流主管勾選）→ 新增或修改時，通知**物流主管**有派遣單待確認
   - 派遣單狀態為「已勾選配送」（已指派給送貨人員，但尚未出發）→ 新增或修改時，同時通知**物流主管**與對應**送貨人員**：資料已更新，路線可能需要重新確認或重新產生
3. **通知管道**：
   - App 內提示（未讀角標／通知列表）
   - LINE 群組推播（沿用既有 LINE Notify／Messaging API 串接）
4. 送貨人員收到通知後，可於 App 重新查看更新後的配送名單，並依 5.3 流程重新調整路線

## 6. 網頁後台功能（新增派遣單匯入）

- 客戶資料建檔／Excel、CSV 匯入
- 人員資料管理（姓名、角色、住家地址、LINE 群組設定）
- 系統設定（公司地址）
- **派遣單匯入與查詢**（新增）：批次匯入 CSV（送貨日期、客戶編號、客戶名稱、住址、電話、產品項目、數量），可依送貨日期查詢派遣單狀態；出發前如有新增／修改，系統自動通知物流主管及相關送貨人員（見 5.4）

不包含每日派工畫面供內勤直接排定路線——物流的「今日配送客戶」由物流主管於 App 勾選，路線則由物流主管或送貨人員於 App 端自行產生／調整，內勤僅負責派遣單資料匯入。

## 7. 路線最佳化邏輯（硬性排序，業務／物流共用同一套邏輯）

1. 將當日勾選的客戶分成「優先」與「一般」兩組
   - 業務模式：依客戶主檔的「優先客戶」欄位
   - 物流模式：依物流主管勾選當下標記的「優先客戶」
2. 優先組內部依最短路徑排序
3. 一般組同樣依最短路徑排序，接在優先組之後
4. 兩組合併為當日完整路線：出發地 → 優先組 → 一般組 → 目的地

不含車輛載重上限、時間窗限制、打卡簽到等功能。

## 8. 技術選型建議（未更動）

| 項目 | 建議方案 |
|---|---|
| 路線距離計算 | Google Maps Distance Matrix API |
| 排序演算法 | 最近鄰居法（Nearest Neighbor），依優先/一般分組排序 |
| LINE 推播 | LINE Notify 或 LINE Messaging API |
| App 本機儲存 | 已排序路線需可離線查看，排序當下（呼叫最佳化）需要網路連線 |

## 9. 開發階段

單一階段（MVP）：

1. 客戶資料建置（手動輸入＋Excel/CSV 匯入）
2. 人員資料（含角色欄位）與系統設定（公司地址、LINE 群組）
3. **派遣單資料結構與網頁後台匯入功能**（新增）
4. 手機 App：
   - App 首頁業務／物流入口選擇
   - 業務模式：出發地/目的地選擇、縣市分類客戶勾選、路線最佳化、離線查看、路線調整、導航串接、LINE 分享
   - 物流模式：
     - 物流主管：派遣單清單查看、今日配送客戶勾選、優先客戶標記
     - 送貨人員：配送名單查看、路線自行調整、離線查看、導航串接、LINE 分享
5. 網頁後台：客戶建檔與匯入、人員資料管理（含角色）、派遣單匯入與查詢
